## [CF1503E. 2-Coloring](https://codeforces.com/contest/1503/problem/E)

### Statement

对 $n\times m$ 的矩形计数有多少种蓝黄染色，使得每行均有蓝色且行内蓝色形成一个连续段，每列均有黄色且列内黄色形成一个连续段。

$n,m\leq 2021$

### Solution

{% folding title="Sol" class="green" open=false %}

不妨视 $n,m$ 同阶。

感觉这个题是比较巧妙的，先套路地尝试一行一行填，但是这样很遗憾地状态数就是 $\mathcal{O}(n^3)$ 的，我努力优化了下确信我会了 $\mathcal{O}(n^4)$，但是大抵是没什么用。

这道题带来的启示是这种有关连续段的限制应当尝试画出图看看长什么样，我们可以发现每种颜色至多有两个连通块（每个格子和周围的四个格子连通），具体来说我们现在先只考虑蓝色（黄色同理），每个蓝色格子一定要和一个上边界格子连通或者下边界格子连通（因为每列上蓝色格子是前缀拼后缀），而第一行和最后一行都只能有一个蓝色连通块。

更进一步地，我们发现蓝色连通块和黄色连通块不能恰有一个，因为第一行和最后一行都至少有一个蓝色，第一列和最后一列都至少有一个黄色，两对格子之间的路径必然有交，不能同时满足连通。

我们先假定蓝色不连通，而且蓝色在每行都有且均为连续的，因此就存在一个索引 $i$ 使得 $i$ 行之前（闭）的蓝色构成一个连通块，$i$ 行之后（开）的蓝色构成一个连通块，且两个之间不连通，于是发现（除去恰好差 1 的情况外）第 $i$ 行和第 $i+1$ 行的黄色就连起来了，进一步可以推得黄色就必须全部连通了。

因此我们枚举 $i$，蓝色是形成了一条最远到 $i$ 的左侧单峰格路和一条最远到 $i+1$ 的右侧单峰格路，枚举下两边峰的位置，前缀和优化下拼接就是 $\mathcal{O}(n^2)$ 的了。

然后我们反过来算下黄色，前面提及只有一种情况是黄色和蓝色都不连通的，特判掉就好了。

{% endfolding %}

## [CF2023E. Tree of Life](https://codeforces.com/contest/2023/problem/E)

### Statement

给定一棵树，构造尽可能少的简单路径覆盖使得每个邻边对都在某个路径中被覆盖了。

$n\leq 5\times 10^5$

### Solution

{% folding title="Sol" class="green" open=false %}

试图剥叶子被正义制裁了。事实上你应当意识到如果你就是纯粹的树上剥叶子的话它不会带来比自底向上更多的好处（因为自底向上就是一种合理的剥叶子顺序），而且还会缺乏一些宏观信息。

其实你直接自底向上贪心就好了，你发现你只需要记录这三个信息：已经结束的路径数、有多少个还可以向上走的路径，其中的最大匹配（匹配指可以合并起来）的路径对数有多少。然后在每个结点处，你要让它的每一对儿子的路径至少匹配一次（如果不够了就补），以及如果它还有父亲的话需要单独再让每个儿子拿出来一个路径向上走（且不被允许和这一层的其他路径匹配）。

合并信息时利用下不同颜色的匹配只受奇偶性以及绝对众数的限制，如果存在绝对众数你可以拿出来点本来已经视作可以匹配的路径对拆出来跟绝对众数匹配。

这个题带来的启示是贪心本质上是一种剪枝搜索，把被偏序的状态删掉，然后剩下的状态如果形式好看的话可以试着压缩状态（例如这题里你本来要记对于每个 $x$ 继续往上走的 $x$ 条路径的方案最少已经结束的路径数，但是你发现这些状态是形如从最大的状态开始每一次合并两个路径并增加一条已经结束的路径数得到，于是你只需要记录最大状态以及能合并多少次。

{% endfolding %}

### [P7246 手势密码](https://www.luogu.com.cn/problem/P7246)

### Statement

给定一棵树，每个结点有一个非负整数权，构造一个简单路径多重集使得每个结点被覆盖的次数恰好是其点权，最小化多重集大小。

$n\leq 3\times 10^6$

### Solution

{% folding title="Sol" class="green" open=false %}

和上一题类似，自底向上贪心，你的状态是已经结束了多少条路径以及还有多少条路径在向上走。

这个题的核心是经过 $u$ 的路径数是确定的，于是如果在不同时存在以 $u$ 为底的路径以及以 $u$ 为最高处的路径的话你增加一条向上走的路径总会导致已经结束的路径数也加一，因此总是不优的（因为你本来就可以直接以一的代价添加一条路径的）。

而如果同时存在以 $u$ 为底和以 $u$ 为最高处的路径的话，你总是能无代价地将它们合并使得能向上走的路径数增加 $1$，因此就贪心地合并完这些即可。

{% endfolding %}

## [特征多项式](https://www.luogu.com.cn/problem/P7776)

### Statement

给定一个矩阵 $A$，求其特征多项式 $p(\lambda)=\det\left(\lambda I-A\right)$。

$n\leq 500$

### Solution

{% folding title="Sol" class="green" open=false %}

考虑 $A$ 和 $P A P^{-1}$ 的特征多项式相等，因为

$$
\begin{aligned}
&\det\left(\lambda I-P A P^{-1}\right)\\\\
=&\det\left(P P^{-1}\lambda I-P A P^{-1}\right)\\\\
=&\det\left(P\lambda I P^{-1}-P A P^{-1}\right)\\\\
=&\det(P\left(\lambda I-A\right)P^{-1})\\\\
=&\det(P)\det\left(\lambda I-A\right)\det(P^{-1})\\\\
=&\det\left(\lambda I-A\right)
\end{aligned}
$$

然后我们希望用做这样的变换把 $A$ 消成简单形式，你发现你不能保证消到上三角，因为用第 $i$ 行去消的话乘以 $P^{-1}$ 会影响到第 $i$ 列，于是你放宽限制，允许 $A_{i+1,i}$ 也非 $0$，这样你用第 $i+1$ 行去消只会影响到第 $i+1$ 列，不会影响消 $i$ 列的效果。

然后你得到了一个矩阵 $A$ 满足 $A_{i,j}$ 只在 $j\geq i-1$ 时可能非 $0$，于是你直接展开行列式的定义式，有效的贡献形如每一段是 $r,l,l+1,\cdots,r-1$，随便递推下也算 $\mathcal{O}(n^3)$ 的。

{% endfolding %}

## [CF1572 E. Polygon](https://codeforces.com/contest/1572/problem/E)

### Statement

给定一个凸多边形，选取一个三角剖分并只保留其中 $k$ 条弦，最大化凸多边形中被弦切出的每个区域的最小面积。

$n\leq 200$。

### Solution

最大化最小值，因此考虑二分。

因为弦两两不交，因此最外层存在一个括号结构，枚举第一个最外层的括号，你发现括号内是子问题，但是问题在于剩下的部分是最外层的区域需要拼接上这边的面积，看起来需要计入状态。但好在你只需要最小化满足每个区域都大于定值的区域数，而你总能以少切一刀的代价使得外层区域合法，因此你只需要记录最少切多少刀使得除了最外面的区域均不小于定值，以及在此基础上最大化剩余面积。

时间复杂度 $\mathcal{O}(n^3\log V)$。

## [CF1152F Neko Rules the Catniverse](https://codeforces.com/contest/1152/problem/F2)

### Statement

有 $n$ 个星球，给定 $k,m$，对满足以下条件的 $p_1,p_2,\cdots,p_k$ 计数：

+ $\forall 1\leq i\leq k,\\\; 1\leq p_i\leq n$

+ $\forall 1\leq i<j\leq k,\\\; p_i\neq p_j$

+ $\forall 1\leq i<k,\\\; p_{i+1}\leq p_i+m$

$1\leq n\leq 10^9,\\\; 1\leq k\leq 12,\\\; 1\leq m\leq 4$

### Solution

{% folding title="Sol" class="green" open=false %}

直接从左往右扫，剩余未合并的链只能有不超过 $m$ 个，只需要记录下端点情况，只有 $2^m$ 种，然后再记录下已经选了多少点，状态数是 $\mathcal{O}(2^mk)$ 的，然后转移与位置无关因此可以矩阵快速幂优化，这样做时间复杂度是 $\mathcal{O}(8^mk^3\log n)$ 的，可以写个 dfs 把不会用到的转移删掉减小常数。

记实际状态数是 $t$。

更进一步优化除了使用 soft-$\mathcal{O}(t^\omega)$ 的矩乘以外也可以考虑求出来特征多项式，这样可以把 $\vec{x}A^n$ 变成求 $\vec{x}f(A)$，其中 $f$ 是不超过 $t-1$ 次的多项式，然后利用向量乘矩阵是 $O(t^2)$ 的，秦九韶算法就可以做到 $\mathcal{O}(t^3)$。

综上，时间复杂度 $\mathcal{O}(8^mk^3+2^mkm\log k\log n)$。

{% endfolding %}

## [CF878E. Numbers on the blackboard](https://codeforces.com/contest/878/problem/E)

### Statement

定义一个序列 $$ 的权值是每次选择相邻两个数 $x,y$ 并改成一个 $x+2y$， $\lvert a\rvert-1$ 次操作后剩下的恰好一个数的最大值。

给定一个长为 $n$ 的整数序列，$q$ 次询问，每次查询 $a$ 的一段区间对应的序列的权值。

$n,q\leq 10^5$

### Solution

{% folding title="Sol" class="green" open=false %}

我们称一个数的贡献被 $\times 2$ 的次数为其合并次数，那首先第一项合并次数一定是 $0$，而其余至少 $1$，那你发现你可以让后面很多个位置都只合并 $1$ 次，因为你可以从前往后合并。

因此我们现在忽略掉第一项，然后只算后面的贡献并最终再乘以 $2$，你现在就不要求要全部合并起来了，那这时候可以有一个简单的贪心是把最后一个正数和前面合并成一个数，直到没有正数或者最后一个正数是第二项，你可以简单地用栈来增量维护这个过程，每次向末尾插入，然后你发现你也可以简单地截断后缀，因为合并是从后往前的，因此写个线段树来维护这个扫描线就可以解决区间询问了。

时间复杂度 $\mathcal{O}((n+q)\log n)$。

{% endfolding %}

## [3rd UCup Stage 13 N. $0100$ Insertion](https://qoj.ac/contest/1812/problem/9489)

### Statement

给定一个长为 $n$ 的 $0,1$ 序列，其中有若干位还未确定。

称一个序列是好的当且仅当每次删除其中一个连续的 $0100$ 可以删空它，计数有多少种填未确定的位置的方案使得得到一个好的序列。

$n\leq 500$

### Solution

{% folding title="Sol" class="green" open=false %}

~~0100 是绫地宁宁的意思~~

手玩下发现 $0100$ 其实不太方便，翻转序列变成删除 $0010$。

首先你发现好的序列一定满足任意两个 $1$ 不相邻，而且在遇到第一个 $1$ 时你肯定是把前面最近两个 $0$ 塞给它匹配，那这时候你知道下一项也是 $0$，你需要决策匹配这个 $0$ 还是后面的，你发现如果此时前面仍有未匹配的 $0$ 的话匹配这一个一定更优（因为 $001\cdots0$ 中间这一段都无法使用在此之前未匹配的 $0$），唯一的问题是现在序列开头就是 $001$ 的话会无法确定。

那你发现你就先存在你还剩 $x$ 个未匹配的 $001$ 和 $y$ 个未匹配的 $0$，那你发现它们的顺序是确定的，一定是先 $x$ 个 $001$ 再 $y$ 个 $0$，因为这些 $001$ 都是作为开头的，因此你得到了一个状态数是 $n^2$ 的自动机，直接对这个 dp 就是 $\mathcal{O}(n^3)$ 的。

{% endfolding %}

## [AGC020 F. Arcs on a Circle](https://atcoder.jp/contests/agc020/tasks/agc020_f)

### Statement

有一个周长为 $C$ 的圆，以及 $n$ 个长为 $l_i$ 的圆弧，将这些圆弧互相独立且等概率随机摆放到某些位置上，求覆盖整个圆的概率。

保证周长与弧长均为整数。

$l_i<C\leq 50,\\\; n\leq 6$

### Solution

{% folding title="Sol" class="green" open=false %}

首先认为选一个最长的弧，认为其起始位置是 $0$，这样就可以破环为链了（因为它是最长的所以从后面环绕覆盖到前面也是没用的）。

然后你发现因为弧长和周长都是整数，你对于每个弧的起始位置的小数部分就只在意相对大小，而在随机实数的情况下若干个数中存在相等的概率是 $0$ 因此可以认为起始位置的小数部分相对大小关系恰好有 $(n-1)!$ 种且每一种的概率都是 $\frac{1}{(n-1)!}$，通过此简化下数轴变成长为 $nC$ 且只有整数下标的了，直接 dp 记录你考虑到哪个位置、最远覆盖到哪个位置、填了哪些弧以及用了哪些排名就行了，你后面的状态是 $2n$ 个 bit 且要求前 $n$ 个中 $1$ 个数和后 $n$ 个 $1$ 个数一样，这其实是 $\binom{2n}{n}=\mathcal{O}(\frac{4^n}{\sqrt n})$ 的状态。

你转移时只需要枚举填了哪个弧（排名是由位置确定的），因此时间复杂度是 $\mathcal{O}(4^nn^{2.5}C^2)$ 的。

{% endfolding %}

## [ARC176 E. Max Vector](https://atcoder.jp/contests/arc176/tasks/arc176_e)

### Statement

你有两个大小 $n$ 的正整数向量 $\vec{a},\vec{b}$，以及 $m$ 个大小为 $n$ 的正整数向量 $\vec{x_i}$，你要决策将这 $m$ 个向量 $\vec{x_i}$ 分别是使 $\vec{a}$ 与其 check $\max$ 还是使 $\vec{b}$ 与其 check $\max$，并最小化最终的 $\vec{a}$ 每一项之和加上 $\vec{b}$ 每一项之和。

$n\leq 10,\\\; \vec{a}_i,\vec{b}_i,\vec{x_i}_j,m\leq 500$

### Solution

{% folding title="Sol" class="green" open=false %}

这个限制是十分难以处理的，于是不知道为什么就该想到网络流了。

其实得知网络流标签后就会了啊！设 $V$ 是最大值，对值的限制就考虑切糕模型，$\vec{a}_i$ 对 $x$ check $\max$ 就是限制其不能小于 $x$，那你让连出这样一张图并加入蓝边就可以进行这个限制：

![](/images/arc176e_1.png)

那你现在是两侧选一个做不小于 $x$ 的限制，因此对 $\vec{b}$ 也反着连一下，然后制造虚点 $p,q$，记 $\vec{a}_i$ 的 $x$ 点是 $u$，$\vec{b}_i$ 的则为 $v$，连 $(s,p,+\infty^\epsilon),(p,q,+\infty),(q,t,+\infty^\epsilon),(u,q,+\infty),(p,v,+\infty)$，这样一定会在两个 $+\infty^\epsilon$ 的边中割掉一个表示选择对另一边 check $\max$，最后再将答案减去 $+\infty^\epsilon m$ 即可。

$+\infty^\epsilon$ 只需要取到 $2nV$ 就够了，因此时间复杂度是 $\mathcal{O}(n^2m(m+V)V)$ 的。

{% endfolding %}
