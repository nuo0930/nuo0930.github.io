    A. 好数

    称一个数 xxx 为好数当且仅当 x∈Nx\in Nx∈N 且将 xxx 转为无前导零的二进制数后存在连续的 333 个 000 或连续的 333 个 111。
    给定 l,rl,rl,r，求 [l,r][l,r][l,r] 间有多少个好数。
    0≤l≤r<2310\leq l\leq r< 2^{31}0≤l≤r<231

    拆成前缀和相减然后数位 dp 状态存后三位即可，时间复杂度 O(log⁡r)O(\log r)O(logr)

    B. MST

    给一个点数为 nnn、边数为 mmm 的无向图，求第 kkk 小生成树。1≤n≤501\leq n\leq 501≤n≤50，1≤m≤30001\leq m\leq 30001≤m≤3000，1≤k≤1041\leq k\leq 10^41≤k≤104

    看到这种比较奇怪的第 kkk 小问题就不难想到 NOI2010 超级钢琴，也是一样的考虑用优先队列维护若干状态每次提取其中最小的然后继续扩展。
    于是我们需要一种状态顺序能使得前面的一定小于等于后面的。
    考虑对所有边进行排序。
    状态记为限制前 ttt 条边是否在生成树，表示在这 ttt 条限制下的最小生成树。不难发现此时若状态 uuu 的限制为状态 vvv 的限制的子集，则 uuu 的边权和一定不大于 vvv 的边权和，满足了有序性。
    于是每个状态用 bitset 记限制，然后再记一维 ttt 表示是限制的前 ttt 条边，用类似 kruskal 的算法计算在限制下最小生成树，然后用优先队列维护，每次扩展时是从 t+1t+1t+1 往后枚举 iii，在生成树上选 [t+1,i)[t+1,i)[t+1,i)，不选 iii，发现最多枚举 O(n)O(n)O(n) 个 iii（因为最多只能选 n−1n-1n−1 条边），所以复杂度 O(nmk+nklog⁡nk)O(nmk+nk\log{nk})O(nmk+nklognk)。

    但是发现错了，因为限制不同是状态表示出的生成树不同的必要条件而非充分条件，所以需要去重。可是限制位数不同也会影响扩展，如果直接在加入优先队列前去重会缺状态，但是如果又全丢进队列只是不计算到排名中复杂度又会变到 O(nm2k)O(nm^2k)O(nm2k)。所以还需要扩展一下前 iii 条边都被限制且前 i−1i-1i−1 条选不选的状态与此时生成树中该边选不选的状态相同、第 iii 选不选的状态不同的情况。

    发现多扩展的这部分也不会影响总复杂度，依然为 O(nmk)O(nmk)O(nmk)，考虑到常数很小所以可过。

    C. string

    有一个长度为 nnn 的 010101 串 sss，其中若干位已知，若干位未知。给定 qqq 条限制形如 s[l1,r1]=s[l2,r2]s[l_1,r_1]=s[l_2,r_2]s[l1​,r1​]=s[l2​,r2​]，求满足限制的字典序最小的 sss。保证有解，1≤n,q≤1061\leq n,q\leq 10^61≤n,q≤106

    cyf：「看我 O(nq)O(nq)O(nq) 冲过 10610^6106」
    于是他是 c·一秒 101210^{12}1012 次·yf 大帝。
    谁敢横刀立马，唯我 cyf 大帝。
    【此处应贴图，但为保护个人肖像权已撤下】

    不难发现我们只关心对于每个 iii，根据限制得到的每个 jjj 是否存在已知为 111 的，如果有则 si=1s_i=1si​=1，否则 si=0s_i=0si​=0。
    但考虑数据范围你明显不能这么做（cyf 大帝例外，因为他一秒 101210^{12}1012 次）。
    题解告诉我，考虑一个类似 st 表的东西。我们维护第 lll 层是 [1,2l],[2,2l+1],⋯ ,[n−2l+1,n][1,2^l],[2,2^l+1],\cdots,[n-2^l+1,n][1,2l],[2,2l+1],⋯,[n−2l+1,n] 这若干个区间的相等关系，然后最后查询就是关心第 000 的相等关系。
    具体来说，由于我们不在意相等关系被重复用几次，所以对于每个限制用类似 st 表的拆分方式拆成某一层两组区间的相等关系，用并查集维护。
    然后在从高层往低层更新，即把 s[a,a+2l+1−1]=s[b,b+2l+1−1]s[a,a+2^{l+1}-1]=s[b,b+2^{l+1}-1]s[a,a+2l+1−1]=s[b,b+2l+1−1] 拆成 s[a,a+2l−1]=s[b,b+2l−1]s[a,a+2^{l}-1]=s[b,b+2^{l}-1]s[a,a+2l−1]=s[b,b+2l−1] 和 s[a+2l,a+2l+1−1]=s[b+2l,b+2l+1−1]s[a+2^l,a+2^{l+1}-1]=s[b+2^l,b+2^{l+1}-1]s[a+2l,a+2l+1−1]=s[b+2l,b+2l+1−1]，更新第 iii 层的关系即可（相等关系是可以被传递的，所以我们只需要枚举 aaa，然后 bbb 取在 l+1l+1l+1 层所属并查集的根就行了）。
    总时间复杂度 O((n+q)log⁡n⋅α(n))O((n+q)\log n \cdot \alpha(n))O((n+q)logn⋅α(n))

    D. tree

    给定一棵大小为 nnn 的树，树上每个点 uuu 均有一个权值 huh_uhu​，qqq 次操作分为两种：

    1. 将 huh_uhu​ 改为 vvv，保证原先的 hu≤vh_u\leq vhu​≤v
    2. 求只考虑树上所有 hu≥vh_u\geq vhu​≥v 的 uuu 时的连通块数
      保证所有 111 号操作中的 vvv 单调非降。
      1≤n,q,hu,v≤1061\leq n,q,h_u,v\leq 10^61≤n,q,hu​,v≤106

    我们定义边 (u,v)(u,v)(u,v) 的权值为 min⁡{hu,hv}\min\{h_u,h_v\}min{hu​,hv​}
    发现 222 号操作问的即为点权不小于 vvv 的点数减去边权不小于 vvv 的边数。
    我们可以把点权和边权都丢到一个权值树状数组中进行维护。
    发现每次修改操作只会修改一个点权，所以可以直接在树状数组中改，但是边权会涉及很多，但我们发现如果这条边的最小值不是来自 uuu 的话那么此次修改对其是没有影响的（因为每次修改只会往大了改），可以跳过。如果原本是且改完后依然是的话也可以统一操作修改。至于剩下的，我们只需要能找到然后暴力改就行，算一下复杂度发现这种情况由于 vvv 单调非降，在两次关于这个点的修改间只会出现一次，复杂度可以摊掉。

    于是我们要做的是快速找到这样的点。考虑先随便定个根，然后对于每个点开个 set （甚至你开个能删除任意节点的堆也行）维护其孩子中点权比它大的点，再单独处理父亲就好了，顺便对父亲的 set 进行更新。时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn)。
